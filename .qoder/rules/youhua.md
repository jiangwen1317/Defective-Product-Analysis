# 代码优化规则

## 无风险重构原则

### 核心要求
- **保持行为一致**：重构后代码必须与原代码产生完全相同的输出和副作用
- **小步前进**：每次只做一个小的改动，验证后再进行下一步
- **有测试保障**：确保有足够的测试覆盖，重构前先运行测试确认通过

### 安全重构技术

#### 1. 提取方法
- 将重复代码块提取为独立函数
- 函数命名清晰表达其职责
- 保持函数单一职责原则

#### 2. 简化条件表达式
- 使用提前返回减少嵌套层级
- 合并重复的条件判断
- 将复杂条件提取为有意义的布尔变量或函数

#### 3. 消除冗余代码
- 移除未使用的变量、导入和函数
- 合并重复的逻辑分支
- 简化冗余的条件判断（如 `if x: return True else: return False` → `return x`）

#### 4. 变量命名优化
- 使用更具描述性的变量名
- 遵循 Python 命名规范（snake_case）
- 避免使用无意义的单字母变量（循环变量除外）

#### 5. 简化数据结构操作
- 使用列表推导式替代简单的 for 循环
- 使用字典的 `get()` 方法处理默认值
- 使用 `with` 语句管理资源

### 禁止操作
- 禁止改变函数的输入输出契约
- 禁止修改算法的核心逻辑
- 禁止删除看似"无用"但实际影响行为的代码（如副作用）
- 禁止在不确定的情况下"优化"性能

### 验证清单
- [ ] 所有现有测试通过
- [ ] 边界条件处理不变
- [ ] 异常处理逻辑不变
- [ ] 副作用（日志、状态修改）不变
- [ ] 代码可读性提升

### 示例

**重构前：**
def get_status(data):
    if data is not None:
        if data.get("active"):
            return True
        else:
            return False
    else:
        return False

**重构后：**
def get_status(data: dict | None) -> bool:
    """获取数据状态。

    Args:
        data: 数据字典，可为空。

    Returns:
        如果数据存在且 active 为 True 则返回 True，否则返回 False。
    """
    return bool(data and data.get("active"))
